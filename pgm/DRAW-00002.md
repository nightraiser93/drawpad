# DRAW-00002 — Live LAN viewing

- **Slice value:** The standout feature — anyone on the same wifi can open a URL in a browser and watch the sketch being drawn live, no app install needed on the viewing device.

## Intent
Stream the phone's live drawing to any browser on the same wifi via an embedded local server. Phone runs an HTTP server (serving a single static viewer page) and a WebSocket server (streaming vector stroke events as they're drawn). Viewer is read-only for this slice.

## Acceptance criteria
- [ ] App hosts a local HTTP server (react-native-tcp-socket) serving a single static viewer page
- [ ] App hosts a WebSocket server (`ws`) streaming vector stroke events live as they're drawn
- [ ] Browser viewer renders incoming stroke events live on an HTML canvas — view-only, no draw-back
- [ ] App displays the local URL plus a QR code so other devices don't need to type the IP
- [ ] URL includes a token so devices on the same wifi can't join by guessing the URL
- [ ] Server starts when a drawing session begins and stops when the app is backgrounded

## Subtasks
- [ ] Wire up react-native-tcp-socket + ws in the Expo dev client build (config plugin)
- [ ] Serve static viewer HTML/JS bundle over HTTP
- [ ] Stream stroke events (from DRAW-00001's stroke data model) over WebSocket
- [ ] Build browser-side canvas renderer for incoming stroke events
- [ ] Add QR code + token generation, display on-screen
- [ ] Start/stop server on session foreground/background
