# DRAW-00002 — Live LAN viewing

- **Slice value:** The standout feature — anyone on the same wifi can open a URL in a browser and watch the sketch being drawn live, no app install needed on the viewing device.

## Intent
Stream the phone's live drawing to any browser on the same wifi via an embedded local server. Phone runs an HTTP server (serving a single static viewer page) and a WebSocket server (streaming vector stroke events as they're drawn). Viewer is read-only for this slice.

## Acceptance criteria
- [x] App hosts a local HTTP server (react-native-tcp-socket) serving a single static viewer page
- [x] App hosts a WebSocket server streaming vector stroke events live as they're drawn (hand-rolled RFC6455 framing on the raw TCP socket — see deviation note below)
- [x] Browser viewer renders incoming stroke events live on an HTML canvas — view-only, no draw-back
- [x] App displays the local URL plus a QR code so other devices don't need to type the IP
- [x] URL includes a token so devices on the same wifi can't join by guessing the URL
- [x] Server starts when a drawing session begins and stops when the app is backgrounded

## Subtasks
- [x] Wire up react-native-tcp-socket in the Expo dev client build (no config plugin needed — autolinked)
- [x] Serve static viewer HTML/JS bundle over HTTP
- [x] Stream stroke events (from DRAW-00001's stroke data model) over WebSocket
- [x] Build browser-side canvas renderer for incoming stroke events
- [x] Add QR code + token generation, display on-screen
- [x] Start/stop server on session foreground/background
